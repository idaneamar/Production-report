ייצוא לחשבשבת — טרי לי
=======================

הקישור (עובד מכל מחשב, ללא התקנה):
https://idaneamar.github.io/Production-report/hashavshevet/

מומלץ להוסיף למחשב המזכירה כסימנייה / קיצור דרך לשולחן העבודה.


מה הכלי עושה
------------
לוקח את דוח הייצור היומי (אותו קובץ xlsx של מחשבון האריזה) ומכין ממנו קובץ
ייבוא לחשבשבת. המזכירה בוחרת לכל לקוח אם להפיק תעודת משלוח / חשבונית / שניהם,
בודקת ומתקנת בטבלה (מק"ט, כמות, חשבון, תאריך), ומורידה את הקובץ לייבוא.


שימוש יומי
----------
1. נכנסים לקישור.
2. גוררים את דוח הייצור (xlsx). בוחרים את היום ואת תאריך המסמך.
3. לכל לקוח בוחרים סוג מסמך: תעודת משלוח / חשבונית / שניהם.
4. בודקים את טבלת העריכה. שורה צהובה = זוהתה ככשרות בד"ץ. אפשר לתקן כל שדה.
5. לוחצים "צור קבצים לחשבשבת" — יורדים שני קבצים: IMOVEIN.DOC ו-IMOVEIN.PRM.


ייבוא בחשבשבת
-------------
1. שומרים את שני הקבצים (IMOVEIN.DOC + IMOVEIN.PRM) בתיקייה:
   \\maya-dt\hash\rep\
   (כדאי לגבות את הקבצים הישנים שם פעם אחת, ליתר ביטחון.)
2. בחשבשבת: ממשקים  ->  יבוא נתוני מלאי  ->  יבוא תנועות למאגר זמני.
3. "יבוא מקובץ טקסט"  ->  בחירת קובץ  ->  IMOVEIN.DOC.
4. "תצוגה מקדימה" — בודקים שהרשומות תקינות (אם יש שגויות, מופיעה סיבת השגיאה).
5. "יבוא".
6. במאגר הזמני בודקים את המסמכים, מתקנים אם צריך, ומאשרים.
   המחיר נשלף אוטומטית בחשבשבת לפי מחירון הלקוח.


מיפויים (לעדכון לפי הצורך)
--------------------------
- לשונית "מיפוי לקוחות": שם הלקוח כפי שמופיע בדוח -> מספר חשבון בחשבשבת.
- לשונית "מיפוי מק"טים": מק"ט שונה לבד"ץ (תא צהוב) ול-P.L. הבסיס נלקח מהדוח.
לקוח בלי מספר חשבון לא ייכלל בקובץ — משלימים אותו בלשונית "מיפוי לקוחות".


קודים (לידיעה)
--------------
סוג מסמך: 20 = תעודת משלוח, 1 = חשבונית מס.
כמות = בדיוק כמו בדוח הייצור (יחידות). מחסן = 1.
